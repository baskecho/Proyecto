INTRODUCCION A LA ARQUITECTURA EMPRESARIAL

  • ¿Qué es arquitectura?
    • La arquitectura es la estructura de algo, en este caso, la empresa.
      • Se está creando
      • Se está planeando
      • Se está modelando
    • Toda arquitectura es diseñada pero todo diseño no es arquitectura.
      • Cuando se está diseñando, se está tomando decisiones y se tiene que asumir las consecuencias, esta es la raíz de las metodologías agiles.
      • Decisiones difíciles de cambiar.
        • Si es difícil se debe poner en el documento de arquitectura
      • Decisiones triviales
        • Hace parte del diseño
      • Una arquitectura no tiene sentido si no se tiene claro su entorno.
    • ISO 42010
      • Conceptos fundamentales o propiedades de un sistema en su entorno encargado en sus elementos, relaciones y en los principios de su diseño y evolución.
    • Arquitectura Monolítica
      • Aquella clásica donde esta una base de datos y un servidor de aplicaciones.
      • Un único punto, si se cae la base de datos o servidor de aplicaciones, se cae la aplicación.
    • Arquitectura Distribuida
      • Servidor descentralizado, evita los inconvenientes de las arquitecturas monolíticas.
    • Ejemplo
      • Sistema Unal tiene gerencia financiera.
      • Existe un interesado
      • Preocupaciones de quien pone los recursos económicos
        • Que sea efectivo el proyecto no debe haber sobrecostos.
          • Un punto de vista, que puede ser un diagrama que le de la información que necesita.
          • Diferencias entre el punto de vista y la vista
            • El punto de vista es un patrón
              • Un diagrama de caso de uno
            • La vista es un modelo
              • Caso de uso crear alumno.
          • No exista sobrecostos
      • Racionales
        • Aquellas ideas que nos permite tomar decisiones en la arquitectura
  • Tipos de Arquitectura
    • Arquitectura empresarial
      • Arquitectura más general de todas.
      • Se presentan dominios
        • Metas de negocios.
          • La tecnología ayuda a manejar estas metas
        • Estrategias de negocios.
        • Programa de proyectos.
          • Estos proyectos impactan uno o más procesos de costos.
        • Procesos
        • Información y datos
          • OLTP
            • Sistemas transaccionales
          • OLAP
            • Sistemas analíticos
          • KPIs
            • Asociados directamente con los procesos
        • Aplicaciones
          • Orientadas a procesos.
          • Existe un flujo que permite hacerlas más flexibles.
          • Tecnología
          • Las aplicaciones deben correr sobre algo, en este caso los servidores.
    • Arquitectura de solución.
    • Arquitectura de dominio.
    • Componentes / Objetos / Plataformas / Infraestructura.

PROCESOS DE NEGOCIO

Se define las reglas para realizar el proyecto

  • Procesos de Negocio
    • Proponer 1 proceso de negocio para una empresa gubernamental que incluye 4 actividades.
    • Para el proyecto modelar un proceso de negocio.
    • Tres o cuatro actividades de negocio.
    • Aplicación Bussines process management(BPM)
      • Es una disciplina que envuelve algunas cambinaciones de modelado, automatización, ejecución, control, medición y optimización de negocio en apoyo de los objetivos.
      • Abarca sistemas, empleado, clientes y socios dentro de la empresa

      • Análisis de requerimientos
